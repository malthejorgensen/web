CHAOS overview

What is CHAOS?

CHAOS is a database for media content. CHAOS does not store the media itself but rather references (URLs) to the media. CHAOS stores rich metadata (XML) about each object. Furthermore CHAOS allows controlled access to the media via a built-in user and folder system. And last but not least; CHAOS provides search of its database via its indexer (Solr).

API versioning

The current version of CHAOS (as of March 2013) is v5.
Tutorials (including this introduction) can be assumed to be written for v5 unless otherwise noted.
CHAOS v5 was released in November 2012. The primary change from v4 to v5 is that output is now guaranteed to be UTF-8.
v6 is currently in development.

For v1-v5 there is no built-in way in CHAOS to obtain version information, and you have to know which version you are dealing with beforehand. In v6 the API will expose which versions are available.

Objects, metadata and files

Each media reference in CHAOS is called an Object. Objects have a number of fields (also called properties). The two most important properties are Files and Metadatas.

  • Files is a list of files (media URLs) associated with the object
  • Metadatas is a list of metadata (XML strings) associated with the object

Every metadata XML-document has a related metadata schema. Furthermore objects can have relations to other objects.

Object retrieval and search

The CHAOS database is indexed in a Apache Solr index, and searching and general retrieval is done through this index. You then search The metadata fields of the objects in the database are specified by their GUID, so somefield_guid:searchstring searching in different fields can be combined with the AND and OR keywords and conditions can be grouped with parenthesis. Retrieving a single object can be done by "searching" on the GUID field: GUID:object_guid and multiple objects can be retrieved by combining guid searches: GUID:object1_guid OR GUID:object2_guid
